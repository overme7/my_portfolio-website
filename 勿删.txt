****************************************************************************************************************************************************************************************************

将所有文件的更改推送到 Git 远程仓库（例如 GitHub），仅仅执行 git push 是不够的。你需要按照以下步骤操作：

1. 添加所有更改到暂存区
使用以下命令将所有修改过的文件添加到 Git 暂存区：

bash
git add .
. 表示当前目录下的所有文件。
如果只想添加特定文件，可以替换为具体文件名，例如：git add src/components/Header.jsx
2. 提交更改
将暂存区的更改提交到本地仓库，并附上提交信息：

bash
git commit -m "Update all files"
-m 后面是提交信息，建议写清楚本次更新的内容，方便后续查看历史记录。
3. 推送到远程仓库
将本地提交推送到远程仓库：

bash
git push
如果是第一次推送，可能需要设置上游分支：
bash
git push -u origin main
（假设默认分支是 main，如果是 master，则替换为 master）

*****************************

首次推送：
如果是第一次推送整个项目，确保远程仓库为空，或者使用：

bash
git push -u origin main

*****************************

完整的命令很麻烦，可以设置 Git 别名来简化操作。例如，在终端中运行：

bash
git config --global alias.acp '!git add . && git commit -m "$1" && git push'
之后，你可以用一条命令完成全部操作：

bash
git acp "Update Header component"

✅ 正确做法：修改文件 → git add → git commit → git push。
每次修改都需要重新走一遍完整的流程，这样才能确保 Git 正确记录和同步你的更改！)

使用 JSX/JavaScript 的注释方式：

方式 1：单行注释（推荐）
jsx
{/* Your WeChat ID */}
{/* <p className="text-sm text-gray-400">Scan or add my WeChat ID directly</p> */}
方式 2：多行注释
jsx
{/*
  //Your WeChat ID
  <p className="text-sm text-gray-400">Scan or add my WeChat ID directly</p>
*/}
方式 3：HTML 风格注释（只适用于纯 HTML，不推荐在 JSX 中使用）
jsx
<!-- //Your WeChat ID -->
<!-- <p className="text-sm text-gray-400">Scan or add my WeChat ID directly</p> -->
在 JSX 中推荐使用方式 1 或方式 2，因为 JSX 需要使用 {/* */} 包裹注释内容

## 第二步：选择托管平台（推荐免费选项）

### 1. **Vercel**（最推荐，Vite 官方推荐）
- 优点：自动部署、国内访问快、支持自定义域名
- 步骤：
  1. 访问 [vercel.com](https://vercel.com)
  2. 用 GitHub/GitLab 账号登录
  3. 导入你的 Git 仓库或拖拽 `dist/` 文件夹
  4. 自动获得 `.vercel.app` 域名

### 2. **Netlify**
- 类似 Vercel，拖拽部署即可
- 支持自定义域名

### 3. **GitHub Pages**（免费但速度一般）
```bash

npm install -D gh-pages

# 在 package.json 添加：
"scripts": {
  "deploy": "gh-pages -d dist"
}

# 部署
npm run deploy
```

### 4. **Cloudflare Pages**（免费+CDN加速）
